## 2. 核心业务逻辑

### 2.1 六合红包业务流程（⭐ 最重要）

#### 业务规则
1. **群组限制**: 只能在固定群组 `69d3f89668f596338b0c1930`（六合天下）发送
2. **时间段控制**:
   - 🟢 **投注时段** (00:00-20:32): 可发红包、可投注
   - 🟡 **封闭等待** (20:33-21:32): 禁止任何操作，显示倒计时
   - 🔵 **结算时段** (21:33-23:59): 系统自动开奖结算

3. **赔率机制**:
   - 理论赔率: 49倍
   - 平台抽成: 1倍
   - 玩家实得: 48倍
   - **单号上限** = 奖池金额 ÷ 48
   - 例如: 奖池 490 USDT → 单号最多投 10.21 USDT

4. **生肖投注**:
   - 支持多选生肖，自动选中对应号码
   - 今年马年，1号=马，13号=马，以此类推
   - 每个生肖对应 4-5 个号码

### 3.2 Axios 请求拦截器

**位置**: `src/api.js`

**功能**:
1. 自动添加 Authorization header
2. 统一处理 401 未授权错误
3. 统一错误提示

```javascript
// 请求拦截器 - 自动添加 token
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// 响应拦截器 - 统一错误处理
api.interceptors.response.use(
  response => {
    // 后端返回格式: {success: true, data: {}, message: ""}
    if (response.data.success !== undefined) {
      return response.data.data
    }
    return response.data
  },
  error => {
    if (error.response?.status === 401) {
      // token 过期，清除本地存储并跳转登录
      localStorage.removeItem('token')
      localStorage.removeItem('user')
      localStorage.removeItem('userId')
      window.location.href = '/login'
    }
    return Promise.reject(error)
  }
)
```

### 3.3 路由守卫

**位置**: `src/main.js`

**功能**: 检查 token 是否过期，保护需要认证的页面

```javascript
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')
  const tokenExpiresAt = localStorage.getItem('tokenExpiresAt')
  const now = new Date().getTime()
  
  const requiresAuth = ['/home', '/contacts', '/wallet', '/profile', '/chat']
  
  if (requiresAuth.includes(to.path)) {
    if (!token || !tokenExpiresAt) {
      next('/login')
      return
    }
    
    if (now > parseInt(tokenExpiresAt)) {
      // token 已过期
      localStorage.clear()
      next('/login')
      return
    }
  }
  
  next()
})
```

**关键点**:
- Token 有效期: 24小时
- 过期时间存储在 `tokenExpiresAt`
- 需要认证的路由列表需手动维护

#### 问题4: 六合红包只有自己看得到（⭐ 重要）
**现象**: 用户A发送六合红包后，只有用户A能看到，用户B看不到

**原因分析**:
1. **前端未监听 `newLiuheRedPacket` 事件** - 最常见
2. **前端未加入正确的 Socket 房间** - 导致后端推送找不到目标
3. **监听器未正确清理** - 导致重复监听或内存泄漏

**解决方案**:

**步骤1: 确保 socket.js 导出监听方法**
```javascript
// socket.js - 添加新方法
export const onNewLiuheRedPacket = (callback) => {
  if (socket) {
    socket.on('newLiuheRedPacket', callback)
  }
}
```

**步骤2: 在组件中正确监听和清理**
```javascript
// LiuHe.vue
import { onNewLiuheRedPacket, joinRoom, initSocket, getSocket } from '../socket'

let newLiuheRedPacketHandler = null

onMounted(async () => {
  // 1. 初始化 Socket
  initSocket()
  
  // 2. 加入群组房间（必须！）
  joinRoom(LIUHE_GROUP_ID)  // '69d3f89668f596338b0c1930'
  
  // 3. 监听新红包事件
  const socket = getSocket()
  newLiuheRedPacketHandler = (data) => {
    console.log('🧧 收到新六合红包:', data)
    
    // 刷新红包列表
    loadRedPackets()
    
    // 显示提示
    if (data.success && data.data) {
      showToast(`${data.data.banker?.username} 发了一个六合红包！`)
    }
  }
  socket.on('newLiuheRedPacket', newLiuheRedPacketHandler)
})

onUnmounted(() => {
  // 4. 清理监听器（防止内存泄漏）
  const socket = getSocket()
  if (socket && newLiuheRedPacketHandler) {
    socket.off('newLiuheRedPacket', newLiuheRedPacketHandler)
    console.log('✅ Socket 监听器已清理')
  }
})
```

### 5.1 统一的错误处理

**推荐做法**:
```javascript
try {
  const response = await walletAPI.getWalletInfo()
  walletInfo.value = response
} catch (error) {
  console.error('获取钱包信息失败:', error)
  
  // 区分错误类型
  if (error.response) {
    // 服务器返回错误
    const status = error.response.status
    const message = error.response.data?.message || '未知错误'
    
    switch (status) {

      case 403:
        showToast('没有权限访问')

        showToast('服务器错误，请稍后重试')
        break
      default:
        showToast(`请求失败: ${message}`)
    }
  } else if (error.request) {
    // 网络错误
    showToast('网络连接失败，请检查网络')
  } else {
    // 其他错误
    showToast('操作失败，请稍后重试')
  }
}
```

### 5.2 Token 刷新策略

**当前实现**: Token 有效期 24 小时，过期后强制重新登录

**改进建议**: 
```javascript
// 建议在响应拦截器中添加 token 续期逻辑
api.interceptors.response.use(
  response => {
    // 检查响应头中是否有新 token
    const newToken = response.headers['x-new-token']
    if (newToken) {
      localStorage.setItem('token', newToken)
      // 更新过期时间
      const expiresAt = new Date().getTime() + 24 * 60 * 60 * 1000
      localStorage.setItem('tokenExpiresAt', expiresAt.toString())
    }
    
    return response.data.data || response.data
  }
)
```

## 6. Socket.io 实时通信

### 6.1 Socket 初始化与管理

**位置**: `src/socket.js`

**单例模式实现**:
```javascript
let socket = null

export const initSocket = () => {
  if (!socket) {
    const token = localStorage.getItem('token')
    socket = io('http://localhost:5000', {
      transports: ['websocket'],  // 强制使用 WebSocket
      reconnection: true,
      reconnectionAttempts: 5,
      reconnectionDelay: 1000,
      auth: { token }
    })
    
    // 连接事件
    socket.on('connect', () => {
      console.log('✅ Socket connected:', socket.id)
    })
    
    socket.on('disconnect', () => {
      console.log('❌ Socket disconnected')
    })
  }
  return socket
}

export const getSocket = () => {
  if (!socket) {
    return initSocket()
  }
  return socket
```

### 6.3 事件监听最佳实践

**问题**: 多次挂载导致重复监听

**解决方案**: 记录监听器引用，卸载时移除

```javascript
import { onMounted, onUnmounted } from 'vue'
import { getSocket } from '../socket'

let messageHandler = null
let redPacketHandler = null

onMounted(() => {
  const socket = getSocket()
  
  // 定义处理函数
  messageHandler = (data) => {
    console.log('收到消息:', data)
    chatMessages.value.push(data)
  }
  
  redPacketHandler = (data) => {
    console.log('收到红包:', data)
    chatMessages.value.push(data)
  }
  
  // 注册监听
  socket.on('groupMessage', messageHandler)
  socket.on('groupRedPacket', redPacketHandler)
})

onUnmounted(() => {
  const socket = getSocket()
  
  // 移除监听（防止内存泄漏）
  if (messageHandler) {
    socket.off('groupMessage', messageHandler)
  }
  if (redPacketHandler) {
    socket.off('groupRedPacket', redPacketHandler)
  }
})
```

### 7.2 数据同步策略

**场景**: 本地缓存 vs 服务器数据

**推荐策略**:
```javascript
const loadData = async () => {
  loading.value = true
  
  try {
    // 1. 先从本地加载（快速显示）
    const localData = getStorageItem('some_key')
    if (localData) {
      dataList.value = localData
    }
    
    // 2. 从服务器获取最新数据
    const serverData = await fetchDataAPI()
    dataList.value = serverData
    
    // 3. 更新本地缓存
    localStorage.setItem('some_key', JSON.stringify(serverData))
    
  } catch (error) {
    console.error('加载数据失败:', error)
    
    // 如果服务器请求失败，至少还有本地数据
    if (dataList.value.length === 0) {
      showToast('加载失败，显示本地缓存数据')
    }
  } finally {
    loading.value = false
  }
}
```
