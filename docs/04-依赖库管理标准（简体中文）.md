# 依赖库管理标准

## 1. 标准概述

为了统一项目中依赖库的管理方式，确保所有技能能够正确使用和添加依赖，特制定本标准。本标准基于项目的实际需求，确保依赖管理的一致性和可靠性。

## 2. 依赖存储规范

### 2.1 存储位置
- **配置文件**：项目根目录下的 `package.json` 文件
- **依赖记录**：项目根目录下的 `yilai/` 文件夹
- **文档位置**：项目根目录下的 `docs/` 文件夹

### 2.2 文件结构

```
project/
├── package.json          # 依赖配置文件
├── yilai/
│   └── dependencies.json # 依赖详细信息
└── docs/
    └── 04-依赖库管理标准（简体中文）.md # 依赖管理标准文档
```

## 3. 依赖类型

### 3.1 生产依赖
- **定义**：项目运行时必需的依赖
- **示例**：playwright、random
- **安装命令**：`npm install <dependency-name>`

### 3.2 开发依赖
- **定义**：开发和构建过程中使用的依赖
- **示例**：@types/node、@types/random
- **安装命令**：`npm install --save-dev <dependency-name>`

## 4. 依赖管理流程

### 4.1 添加新依赖

1. **评估需求**：确定是否真的需要添加新依赖
2. **选择依赖**：选择合适的依赖库，考虑稳定性、活跃度和维护状态
3. **安装依赖**：运行 `npm install <dependency-name>` 安装依赖
4. **更新文档**：
   - 更新 `yilai/dependencies.json` 文件，添加新依赖的详细信息
   - 记录依赖的用途和使用位置
5. **测试验证**：确保新依赖能够正常工作，不会与现有依赖冲突

### 4.2 更新依赖

1. **检查更新**：定期运行 `npm outdated` 检查依赖更新
2. **评估更新**：评估更新的必要性和风险
3. **执行更新**：运行 `npm update <dependency-name>` 更新依赖
4. **测试验证**：确保更新后的依赖能够正常工作
5. **更新文档**：更新 `yilai/dependencies.json` 文件中的版本信息

### 4.3 删除依赖

1. **评估必要性**：确定依赖是否真的不再需要
2. **执行删除**：运行 `npm uninstall <dependency-name>` 删除依赖
3. **清理代码**：移除代码中对该依赖的引用
4. **更新文档**：从 `yilai/dependencies.json` 文件中移除该依赖的信息

## 5. 依赖记录规范

### 5.1 dependencies.json 格式

```json
{
  "name": "project-dependencies",
  "version": "1.0.0",
  "description": "项目依赖说明书",
  "lastUpdated": "YYYY-MM-DD",
  "dependencies": [
    {
      "name": "dependency-name",
      "version": "^x.x.x",
      "type": "production|development",
      "description": "依赖的详细描述",
      "installation": "npm install dependency-name",
      "usedBy": [
        "path/to/file1.js",
        "path/to/file2.ts"
      ]
    }
  ],
  "installationGuide": {
    "title": "依赖安装指南",
    "steps": [
      "步骤1",
      "步骤2"
    ],
    "addingNewDependency": "添加新依赖的步骤"
  },
  "notes": [
    "注意事项1",
    "注意事项2"
  ]
}
```

### 5.2 记录内容

对于每个依赖，必须记录以下信息：
- **name**：依赖的名称
- **version**：依赖的版本
- **type**：依赖的类型（production 或 development）
- **description**：依赖的详细描述和用途
- **installation**：安装命令
- **usedBy**：使用该依赖的文件路径列表

## 6. 安装指南

### 6.1 首次安装

1. 确保已安装Node.js（推荐版本16.0.0及以上）
2. 打开终端，进入项目根目录
3. 运行命令：`npm install`，安装所有依赖
4. 对于playwright，还需要运行：`npx playwright install`，安装浏览器驱动

### 6.2 环境要求

- **Node.js**：推荐版本16.0.0及以上
- **npm**：推荐版本8.0.0及以上
- **操作系统**：Windows、macOS、Linux

## 7. 最佳实践

### 7.1 依赖选择
- 选择稳定、活跃维护的依赖库
- 优先选择轻量级的依赖
- 避免过多的依赖，减少项目的复杂性

### 7.2 版本管理
- 使用语义化版本号
- 在package.json中锁定依赖版本
- 定期更新依赖，确保安全性和稳定性

### 7.3 文档管理
- 保持依赖文档的更新
- 记录依赖的用途和使用位置
- 提供清晰的安装和使用指南

### 7.4 安全性
- 定期检查依赖的安全漏洞
- 避免使用有已知安全问题的依赖
- 及时更新存在安全问题的依赖

## 8. 故障排除

### 8.1 依赖安装失败

- 检查网络连接是否正常
- 尝试使用 `npm cache clean --force` 清理缓存
- 确保Node.js版本符合要求
- 尝试使用 `npm install --verbose` 查看详细错误信息

### 8.2 依赖版本冲突

- 使用 `npm ls` 查看依赖树，找出冲突的依赖
- 尝试使用 `npm dedupe` 解决依赖冲突
- 必要时指定特定版本的依赖
- 考虑使用 `npm shrinkwrap` 或 `package-lock.json` 锁定依赖版本

### 8.3 依赖运行时错误

- 检查依赖版本是否与代码兼容
- 查看依赖的官方文档，了解正确的使用方法
- 尝试降级依赖版本，看看是否能够解决问题
- 检查是否缺少必要的依赖项

## 9. 依赖使用示例

### 9.1 Playwright

**用途**：浏览器自动化，用于模拟用户操作

**安装**：`npm install playwright`

**使用示例**：

```typescript
import { chromium, Page, Browser } from 'playwright';

// 启动浏览器
const browser = await chromium.launch({
  headless: false,
  args: ['--disable-notifications']
});

// 创建页面
const page = await browser.newPage();

// 访问网站
await page.goto('https://www.facebook.com');
```

### 9.2 Random

**用途**：随机数生成，用于随机选择关键词

**安装**：`npm install random @types/random`

**使用示例**：

```typescript
import random from 'random';

// 随机选择一个关键词
const keyword = keywords[random.int(0, keywords.length - 1)];
```
