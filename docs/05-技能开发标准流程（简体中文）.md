# 技能开发标准流程

## 1. 标准概述

为了统一项目中技能的开发流程，确保所有技能能够按照一致的标准进行开发和集成，特制定本标准。本标准基于项目的实际需求，确保技能开发的一致性和可靠性。

## 2. 技能目录结构

### 2.1 基本结构

```
skills/
├── {platform}/           # 按平台分类的技能目录
│   ├── {platform}-skills.ts    # 平台相关技能实现
│   └── mcp-{platform}-*.skill.json  # 技能元数据文件
├── {skill-name}.ts      # 独立技能实现
├── {skill-name}.skill.json  # 技能元数据文件
└── test-{skill-name}.ts # 技能测试脚本
```

### 2.2 示例

```
skills/
├── weibo/
│   ├── weibo-skills.ts
│   ├── mcp-weibo-post.skill.json
│   └── mcp-weibo-follow.skill.json
├── facebook/
│   ├── facebook-skills.ts
│   └── mcp-facebook-post.skill.json
├── facebook-join-groups.ts
├── facebook-join-groups.skill.json
└── test-facebook-join-groups.ts
```

## 3. 技能开发流程

### 3.1 步骤 1：定义技能元数据

创建 `{skill-name}.skill.json` 文件，包含以下字段：

```json
{
  "skillId": "mcp-weibo-{skill-name}",
  "name": "{skill-name}",
  "version": "1.0.0",
  "description": "技能描述",
  "inputSchema": {
    "type": "object",
    "properties": {
      "param1": {
        "type": "string",
        "description": "参数1描述"
      },
      "param2": {
        "type": "number",
        "default": 1,
        "description": "参数2描述"
      }
    }
  },
  "outputSchema": {
    "type": "object",
    "properties": {
      "result": {
        "type": "string",
        "description": "结果描述"
      }
    }
  },
  "healthCheckUrl": "/health/{skill-name}",
  "communicationMode": "local-function",
  "timeout": 300000
}
```

### 3.2 步骤 2：实现技能逻辑

创建 `{skill-name}.ts` 文件，实现技能的核心逻辑：

```typescript
// 导入必要的依赖
import { chromium, Page, Browser } from 'playwright';
import * as fs from 'fs';

// 定义输入输出接口
interface SkillInput {
  param1?: string;
  param2?: number;
  traceId?: string;
  taskId?: string;
  accountId?: string;

interface SkillOutput {
  result: string;
}

interface SkillResult {
  ok: boolean;
  code: number;
  message: string;
  data?: SkillOutput;
  traceId?: string;
  durationMs?: number;
}

class SkillName {
  // 实现技能逻辑
  async run(input: SkillInput): Promise<SkillResult> {
    const startTime = Date.now();
    const traceId = input.traceId || `trace-${Date.now()}`;

    try {
      // 技能逻辑实现
      // ...

      return {
        ok: true,
        code: 0,
        message: '操作成功',
        data: {
          result: '成功结果'
        },
        traceId,
        durationMs: Date.now() - startTime
      };
    } catch (error) {
      return {
        ok: false,
        code: 500,
        message: `操作失败: ${error instanceof Error ? error.message : String(error)}`,

        durationMs: Date.now() - startTime

    }

  healthCheck(): { ok: boolean; message: string } {
    return {
      ok: true,
      message: '技能健康状态正常'
    };
  }
}

// 导出技能
const skillName = new SkillName();

export const skillId = 'mcp-weibo-skill-name';
export const name = 'skill-name';
export const version = '1.0.0';
export const description = '技能描述';
export const inputSchema = {
  type: 'object',
  properties: {
    param1: {
      type: 'string',
      description: '参数1描述'
    },
    param2: {
      type: 'number',
      default: 1,
      description: '参数2描述'
    }
  }
};
export const outputSchema = {
  type: 'object',
  properties: {
    result: {
      type: 'string',
      description: '结果描述'
    }
  }
};
export const healthCheckUrl = '/health/skill-name';
export const communicationMode = 'local-function';
export const timeout = 300000;

// 导出执行函数
export async function execute(input: SkillInput): Promise<SkillResult> {
  return await skillName.run(input);
}

// 导出健康检查函数
export function healthCheck(): { ok: boolean; message: string } {
  return skillName.healthCheck();
}
```

### 3.3 步骤 3：创建测试脚本

创建 `test-{skill-name}.ts` 文件，用于测试技能的功能：

```typescript
import { execute } from './{skill-name}';

async function testSkill() {
  console.log('开始测试技能...');

  try {
    // 测试输入
    const input = {
      param1: '测试参数1',
      param2: 1,
      traceId: `test-${Date.now()}`,
      taskId: `task-${Date.now()}`
    };

    console.log('执行技能操作...');
    const result = await execute(input);

    console.log('执行结果:', JSON.stringify(result, null, 2));

    if (result.ok) {
      console.log('测试成功！');
    } else {
      console.log('测试失败:', result.message);
    }
  } catch (error) {
    console.error('测试过程中出现错误:', error);
  }
}

// 运行测试
testSkill();
```

### 3.5 步骤 5：集成到主骨架

在主骨架文件中集成技能：

```typescript
import { execute as executeSkillName } from './skills/{skill-name}';

// 在路由或工作流中调用技能
app.post('/skill/skill-name', async (req, res) => {
  try {
    const result = await executeSkillName(req.body);
    res.json(result);
  } catch (error) {
    res.status(500).json({
      ok: false,
      code: 500,
      message: `执行失败: ${error instanceof Error ? error.message : String(error)}`
    });
  }
});
```

## 4. 技能开发规范

### 4.1 命名规范

- **技能ID**：`mcp-weibo-{skill-name}`，全局唯一
- **文件命名**：使用小写字母和连字符，例如 `facebook-join-groups.ts`
- **类名**：使用 PascalCase，例如 `FacebookJoinGroups`
- **函数名**：使用 camelCase，例如 `executeSkill`
- **变量名**：使用 camelCase，例如 `inputParam`

### 4.2 代码风格

- 使用 TypeScript 编写技能
- 遵循 ESLint 代码风格
- 使用 async/await 处理异步操作
- 适当添加注释，说明关键逻辑

### 4.3 错误处理

- 使用 try/catch 捕获异常
- 统一返回错误格式：`{ ok: false, code, message, traceId, durationMs }`
- 错误码使用 `SkillErrorCode` 中定义的标准错误码

### 4.4 日志记录

- 使用结构化日志
- 记录关键操作和错误信息
- 包含 traceId 和 taskId 以便追踪

### 4.5 性能优化

- 合理使用异步操作
- 避免不必要的网络请求
- 适当使用缓存

## 5. 技能测试标准

### 5.1 测试类型

- **单元测试**：测试技能的核心函数
- **集成测试**：测试技能与主骨架的集成
- **端到端测试**：测试完整的技能执行流程

### 5.2 测试要求

- 每个技能必须有测试脚本
- 测试脚本应覆盖主要功能和边界情况
- 测试脚本应能够独立运行

### 5.3 测试执行

```bash
# 编译 TypeScript
tsc

# 运行测试脚本
node dist/skills/test-{skill-name}.js
```

## 6. 技能部署标准

### 6.1 部署流程

1. 编译 TypeScript：`tsc`
2. 确保所有依赖已安装：`npm install`
3. 启动服务：`npm run start`

### 6.2 环境要求

- Node.js 16.0.0 及以上
- npm 8.0.0 及以上
- 必要的依赖库

### 6.3 健康检查

技能应提供健康检查接口，确保技能能够正常运行：

```typescript
export function healthCheck(): { ok: boolean; message: string } {
  return {
    ok: true,
    message: '技能健康状态正常'
  };
}
```

## 7. 技能版本管理

### 7.1 版本号规范

使用语义化版本号：`major.minor.patch`

- **major**：不兼容的API变更
- **minor**：向后兼容的功能添加
- **patch**：向后兼容的错误修复

### 7.2 版本更新流程

1. 更新技能实现
2. 更新 `{skill-name}.skill.json` 文件中的版本号
3. 运行测试，确保功能正常
4. 提交代码，标注版本变更

## 8. 技能文档标准

### 8.1 文档内容

每个技能应包含以下文档：

- **技能说明**：技能的功能和用途
- **输入参数**：详细的输入参数说明
- **输出结果**：详细的输出结果说明
- **错误处理**：可能的错误码和处理方法
- **使用示例**：技能的使用示例

### 8.2 文档格式

使用 Markdown 格式编写文档，放置在技能目录中或 docs 目录中。

## 9. 技能安全标准

### 9.1 安全要求

- 不硬编码密钥和敏感信息
- 不记录敏感信息到日志
- 合理处理用户输入，防止注入攻击
- 遵循项目的安全标准

### 9.2 权限管理

- 技能应遵循最小权限原则
- 只访问必要的资源和API

## 10. 技能监控标准

### 10.1 监控指标

- 技能执行时间
- 技能执行成功率
- 技能错误率
- 技能调用次数

### 10.2 监控实现

- 使用 trace/span 记录技能执行过程
- 记录关键操作和错误信息
- 提供技能执行的统计信息

## 11. 技能最佳实践

1. **单一责任**：每个技能只负责一个具体的功能
2. **清晰的输入输出**：明确定义技能的输入和输出
3. **错误处理**：合理处理错误，返回统一的错误格式
4. **日志记录**：适当记录技能执行过程
5. **性能优化**：优化技能执行性能
6. **可测试性**：确保技能可以被测试
7. **可维护性**：编写清晰、可维护的代码
8. **文档完备**：提供完整的技能文档
