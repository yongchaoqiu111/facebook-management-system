# 02-程序员使用说明（简体中文）

> 面向开发者：拿到仓库后，如何读代码、加技能、跑流程、做排障。

---

## 1. 目录导航（建议先看）

- 服务入口：`index.ts`
- 配置中心：`config.ts`
- 技能注册：`skill-registry.ts`
- 编排核心：`task-orchestrator.ts`
- 浏览器驱动：`weibo-browser-automation.ts`
- 智能关注工作流：`smart-follow-workflow.ts`
- 每日发帖工作流：`daily-publish-workflow.ts`
- 技能实现集合：`skills/weibo-skills.ts`
- 标准契约：`standards/contracts.ts`

---

## 2. 环境准备

### 2.1 安装

```bash
npm install
```

### 2.2 配置

1. 复制并编辑 `.env`（可参考 `.env.example`）
2. 填写 LLM 密钥文件：`data/llm_api_key.txt`

最小配置示例：

```dotenv
APP_PORT=3000
LLM_BASE_URL=https://api.scnet.cn/api/llm/v1
LLM_MODEL=Qwen3-235B-A22B
LLM_TIMEOUT_MS=30000
LLM_API_KEY_FILE=data/llm_api_key.txt
```

`data/llm_api_key.txt` 文件只放一行 key。

## 3. 启动与验证

```bash
npm run build
npm run start
```

## 4. 常用接口

### 4.1 每日工作流

- 预览：`POST /workflow/daily/preview`
- 发布：`POST /workflow/daily/publish`

### 4.2 关注工作流

- 基础关注：`POST /workflow/follow/run`
- 智能关注：`POST /workflow/follow/smart-run`

智能关注请求体建议：

```json
{
  "keywords": ["AI", "科技"],
  "maxTotalPosts": 10,
  "perKeywordMaxPosts": 5,
  "maxLikes": 2,
  "maxFollows": 1,
  "maxComments": 1,
  "commentText": "观点很有启发，感谢分享。",
  "loginTimeoutSeconds": 180
}
```

### 4.3 图文/视频发布技能

- 路由：`POST /weibo/post/media/headed`

示例请求体：

```json
{
  "text": "测试图文/视频发布技能（标准化接入）",
  "imagePaths": ["images/1.png"],
  "videoPath": "video/1.MP4",
  "publish": false,
  "loginTimeoutSeconds": 180
}
```

说明：

- `publish=false`：仅填充并上传，不真正点击发布（建议先验收）
- `publish=true`：执行真实发布动作

### 4.4 全自动账号密码登录（无人工扫码）

- 路由：`POST /weibo/login/password/headed`
- 读取优先级：请求体 > `.env` (`WEIBO_LOGIN_USERNAME`) + 密码文件 (`WEIBO_PASSWORD_FILE`)

示例请求体：

```json
{
  "timeoutSeconds": 240
}
```

说明：

- 该流程不经过大模型，仅脚本调用浏览器登录页
- 若触发微博二次验证（验证码/安全校验），接口会返回超时失败
- 建议在失败时回退到扫码登录，保证流程可恢复

## 5. 如何新增一个技能（标准步骤）

1. 在 `skills/` 或独立 workflow 文件实现能力函数
2. 输入输出遵循 `standards/contracts.ts`
3. 在主服务写入对应路由（或编排调用）
4. 在技能目录生成 `*.skill.json`
5. 本地 build + 调试脚本验证

建议每个技能至少具备：

- 参数校验
- 超时控制
- 错误码映射
- 结构化日志

---

## 6. 如何读主骨架调用链

以“每日发布”为例：

1. `index.ts` 定时触发或 API 触发
2. 进入 `daily-publish-workflow.ts`
3. 调用 `skills/weibo-skills.ts` 获取新闻 + 生成文案
4. 调用 `weibo-browser-automation.ts` 执行微博发布

以“智能关注”为例：

1. `index.ts` 接收 `/workflow/follow/smart-run`
2. 进入 `smart-follow-workflow.ts` 做 AI 评估
3. 调用 `weibo-browser-automation.ts` 搜索/点赞/关注/评论
4. 洞察落盘到 `data/tmp/smart_follow_insights.json` 和 `.md`

## 7. 排障手册

### 7.1 LLM 没生效

检查：

- `.env` 中 `LLM_BASE_URL` 是否正确
- `data/llm_api_key.txt` 是否有 key
- 启动日志是否有“LLM未配置完整”告警

### 7.2 浏览器操作不执行

检查：

- 是否登录微博（cookies/state 文件是否更新）
- 页面是否命中可点击元素（微博 DOM 可能变动）
- `maxLikes/maxFollows/maxComments` 是否设为 0

### 7.3 任务重试与失败

- 查看 `task-orchestrator.ts` 中重试逻辑
- 观察 trace/span 与错误码

---

## 8. 推荐工程实践

1. 技能函数保持纯业务，编排逻辑放主骨架
2. 统一返回结构，避免“每个技能一套格式”
3. 使用文本文件管理客户可编辑配置（关键词、人设、LLM key）
4. 所有可调参数都暴露到配置层，不写死在代码中

---

## 9. 贡献约定

提交前至少执行：

```bash
npm run build
```

变更技能接口时，必须同步更新：

- `*.skill.json`
- 文档（本目录）
- 调试脚本（如有）
