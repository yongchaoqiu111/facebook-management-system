# 01-工程接口与技能标准（简体中文）

> 目标：将主骨架与技能生态的“数据/通讯/依赖/驱动/调用方式”统一成可扩展标准。  
> 版本：`v1.0`（当前工程落地版本）

---

## 1. 技术栈标准

- 语言：TypeScript（Node.js 运行时）
- HTTP 服务：Express
- 调度：node-cron
- 队列：BullMQ + Redis
- 缓存/本地存储：SQLite（LLM 缓存）+ 文件系统（配置/洞察）
- 浏览器自动化驱动：Playwright（有头/无头可切换）
- 外部调用：Axios（OpenAI-compatible API）

---

## 2. 分层架构标准

### 2.1 主骨架（Orchestrator）职责

1. 任务编排：串联技能执行顺序
2. 任务治理：幂等锁、重试、死信、人工介入
3. 可观测：trace/span 记录
4. 调度：定时触发任务
5. 配置中心：统一读取 `.env` + 文本配置

### 2.2 技能模块（Skill）职责

1. 单一能力（单责任）
2. 明确输入输出 schema
3. 不侵入主流程控制（由主骨架编排）
4. 错误码与重试语义明确

---

## 3. 技能元数据标准

技能描述文件约定：`skills/*.skill.json`。

**必填字段**

- `skillId`：全局唯一，推荐 `mcp-weibo-*`
- `name`：技能名称
- `version`：语义化版本
- `description`：技能说明
- `inputSchema`：输入 JSON Schema（简化版）
- `outputSchema`：输出结构说明
- `healthCheckUrl`：健康检查地址
- `communicationMode`：通讯模式（`local-function` / `http` / `queue`）
- `timeout`：毫秒超时

契约类型定义见 [standards/contracts.ts](../standards/contracts.ts)。

---

## 4. 数据协议标准（主骨架 ⇄ 技能）

统一请求上下文（建议所有技能执行都携带）：

- `traceId`：全链路追踪ID
- `taskId`：任务ID
- `accountId`：账号ID（可选）
- `requestedAt`：请求时间戳

统一响应信封：

- 成功：`{ ok: true, code: 0, message, data, traceId, durationMs }`
- 失败：`{ ok: false, code, message, errorType?, retryable?, traceId, durationMs }`

错误码建议复用 [standards/contracts.ts](../standards/contracts.ts) 中 `SkillErrorCode`。

---

## 5. 通讯模式标准

### 5.1 local-function（当前主用）

- 适合同仓库内技能
- 延迟低，调试方便
- 由主骨架直接调用技能函数

### 5.2 http

- 适合跨服务部署技能
- 要求：超时、重试、幂等键、鉴权

### 5.3 queue

- 适合耗时任务与批处理
- 要求：消息去重、可重放、可回溯

---

## 6. 依赖治理标准

1. LLM Key 统一来源：`data/llm_api_key.txt`（可用 `LLM_API_KEY_FILE` 覆盖）
2. 禁止技能自行硬编码密钥
3. 所有外部依赖必须超时可配（`timeoutMs`）
4. 文件输出路径统一走 `data/tmp` 或配置项
5. 依赖文档：如需安装额外依赖（如 Python、第三方库等），请在 `yilai` 文件夹中提供安装说明文档
6. 依赖管理：每个技能的依赖应在其技能定义文件中注明，便于统一管理

---

## 7. 驱动调用标准（浏览器自动化）

1. 登录态统一复用（storage state + cookies）
2. 登录检测失败自动回退扫码登录
3. 业务动作必须可限流/限量（例如：最大扫描、点赞、关注、评论）
4. 业务结果需输出结构化统计（总数 + 分关键词）
5. Cookie 管理：所有技能的 Cookie 文件统一存放在 `cookie` 文件夹中，命名格式为 `{platform}.txt`（例如：`facebook.txt`）
6. 账号管理：账号信息统一存放在 `zhanghao` 文件夹中，每个平台一个配置文件，包含账号相关信息
7. Cookie 使用：技能在执行时应优先使用 `cookie` 文件夹中的对应平台 Cookie 文件，如不存在则进行登录流程

---

## 8. 可观测与审计标准

1. 每步技能执行写 span：`stepId` + `skillId` + `status` + `duration`
2. 每个任务有 `traceId`
3. 关键变更落文件（如洞察摘要）并可回放
4. 错误场景区分：可重试 / 不可重试 / 人工介入

---

## 9. 新技能接入标准流程

1. 先定义 I/O 契约（输入参数 + 输出结构 + 错误码）
2. 编写技能实现（单一职责）
3. 注册 `*.skill.json`
4. 在主骨架增加路由或编排步骤
5. 接入 trace + 超时 + 重试策略
6. 补充最小可运行调试脚本

---

## 10. 版本与兼容标准

- `skillId` 不变，`version` 递增
- 输入字段新增：向后兼容（可选字段）
- 输入字段删除或语义变更：必须升级主版本并提供迁移说明

---

## 11. 安全标准（最低要求）

1. `.env` 与密钥文件必须加入 `.gitignore`
2. 日志中不得明文打印 API Key
3. HTTP 模式下建议启用签名鉴权
4. 生产环境禁用调试端点
