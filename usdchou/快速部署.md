# 🚀 USDCHOU 快速部署指南

## 一、购买服务器（5分钟）

**推荐配置：**
- 腾讯云/阿里云：4核8G，100GB SSD，5-10Mbps带宽
- 系统：Ubuntu 20.04 LTS 或 22.04 LTS
- 费用：¥400-600/月

**获取信息：**
- 记录服务器公网IP
- 记录SSH登录密码或密钥

---

## 二、一键部署（10分钟）

### 1. 登录服务器
```bash
ssh root@你的服务器IP
```

### 2. 运行部署脚本
```bash
# 下载脚本
curl -o deploy.sh https://raw.githubusercontent.com/your-repo/usdchou/main/deploy.sh

# 或者直接复制 deploy.sh 内容到服务器

# 执行
chmod +x deploy.sh
./deploy.sh
```

### 3. 上传项目代码
```bash
# 方法1：Git克隆
cd /opt/usdchou
git clone <你的仓库地址> .

# 方法2：SCP上传（在本地电脑执行）
scp -r ./usdchou root@服务器IP:/opt/usdchou/
```

### 4. 安装依赖
```bash
cd /opt/usdchou
npm install
```

### 5. 配置环境变量
```bash
cp .env.example .env
vim .env
```

**必须修改的配置：**
```env
JWT_SECRET=<生成随机字符串>
TRON_API_KEY=<从trongrid.io获取>
```

生成JWT_SECRET：
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 6. 配置Nginx
```bash
# 复制配置文件
cp nginx.conf.example /etc/nginx/sites-available/usdchou

# 编辑配置，修改域名
vim /etc/nginx/sites-available/usdchou

# 启用配置
ln -s /etc/nginx/sites-available/usdchou /etc/nginx/sites-enabled/

# 测试并重载
nginx -t
systemctl restart nginx
```

### 7. 启动应用
```bash
cd /opt/usdchou
pm2 start server.js --name usdchou-backend
pm2 startup systemd
pm2 save
```

### 8. 配置SSL（可选但推荐）
```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d your-domain.com
```

## 三、验证部署

### 检查服务状态
```bash
# 后端
pm2 status

# Nginx
systemctl status nginx

# MongoDB
systemctl status mongod

# Redis
systemctl status redis-server
```

### 访问测试
- 浏览器访问：`http://你的服务器IP`
- API测试：`http://你的服务器IP/api/health`

---

## 四、常用命令

### 查看日志
```bash
pm2 logs usdchou-backend      # 后端日志
tail -f /var/log/nginx/usdchou-access.log   # Nginx访问日志
tail -f /var/log/nginx/usdchou-error.log    # Nginx错误日志
```

### 重启服务
```bash
pm2 restart usdchou-backend   # 重启后端
systemctl restart nginx       # 重启Nginx
```

### 更新代码
```bash
cd /opt/usdchou
git pull
npm install
pm2 restart usdchou-backend
```

### 数据库备份
```bash
# 手动备份
/opt/scripts/backup-mongodb.sh

# 查看备份文件
ls -lh /opt/backups/mongodb/
```

## 五、故障排查

### 问题1：端口被占用
```bash
lsof -i :5000
kill -9 <PID>
```

### 问题2：MongoDB连接失败
```bash
systemctl status mongod
tail -f /var/log/mongodb/mongod.log
```

### 问题3：Nginx 502错误
```bash
pm2 status              # 检查后端是否运行
pm2 logs                # 查看后端日志
nginx -t               # 检查Nginx配置
```

### 问题4：内存不足
```bash
free -h                # 查看内存使用
top                    # 查看进程占用
```
