- 禁用点击打开按钮

**用途**：

### 1. 群聊消息存储模型 ❓
**问题**：当前使用 `GroupMessage` 模型还是 `Message` 模型存储群聊消息？

**影响**：
- 决定 `GET /api/chats/messages/:chatId` 的查询逻辑
- 决定 `POST /api/chats/messages` 的保存逻辑
- 影响数据库索引设计

**需要确认**：
```javascript
// 方案A：统一使用 Message 模型
{
  _id: ObjectId,
  type: 'group' | 'private',  // 消息类型
  groupId: ObjectId,           // 群聊时必填
  receiverId: ObjectId,        // 私聊时必填
  content: String,
  sender: ObjectId,
  createdAt: Date
}

// 方案B：分离 GroupMessage 和 PrivateMessage
GroupMessage {
  _id: ObjectId,
  groupId: ObjectId,
  content: String,
  sender: ObjectId,
  createdAt: Date
}

PrivateMessage {
  _id: ObjectId,
  senderId: ObjectId,
  receiverId: ObjectId,
  content: String,
  createdAt: Date
}
```

**建议**：采用方案A（统一模型），便于维护和扩展
