## ⚠️ 风险与应对

### 风险 1: 后端延期
**概率**：中  
**影响**：高  
**应对**：
- 提前 1 天开始联调准备（前端可以先写 Mock 数据）
- 如果延期，优先保证核心功能（发消息、收消息）
- 非核心功能（图片、搜索）推迟到下一迭代

### 风险 2: 并发问题导致超发
**概率**：低  
**影响**：极高  
**应对**：
- 严格测试 50 人并发场景
- 监控数据库 `remainCount` 和 `totalClaimed`
- 发现超发立即回滚并冻结相关账户

### 风险 3: WebSocket 大规模断连
**概率**：低  
**影响**：高  
**应对**：
- 前端实现指数退避重连
- 后端增加心跳检测
- 准备降级方案（HTTP 轮询）

### 风险 4: 性能不达标
**概率**：中  
**影响**：中  
**应对**：
- 提前进行压力测试
- 准备水平扩展方案（增加服务器）
- 优化数据库索引和查询
