# Day 3 验收标准

**日期**：2026-04-10  
**验收人**：AI 助手 + 项目负责人  
**目标**：确保所有任务完成且质量达标

---

## ✅ 功能验收清单

### 核心功能（必须全部通过）

#### 1. 消息发送与接收
- [ ] 用户 A 发送消息，立即显示在聊天列表
- [ ] 用户 B 实时收到消息（延迟 < 100ms）
- [ ] 消息内容、时间、发送者显示正确
- [ ] 无报错、无警告

**测试方法**：
```
1. 打开 2 个浏览器窗口
2. 登录不同账号
3. 进入同一个接龙群
4. 互相发送消息
5. 观察是否实时显示
```

---

#### 2. 历史消息加载
- [ ] 刷新页面后，历史消息不丢失
- [ ] 从 LocalStorage 加载（快速）
- [ ] LocalStorage 缺失时，从 HTTP 加载
- [ ] 消息顺序正确（旧的在上，新的在下）

**测试方法**：
```
1. 发送 5 条消息
2. 刷新页面
3. 检查 5 条消息是否都显示
4. 检查顺序是否正确
```

---

#### 3. 消息去重
- [ ] WSS 和 HTTP 不重复显示同一条消息
- [ ] 快速刷新页面不重复
- [ ] Set 大小限制生效（无内存泄漏）

**测试方法**：
```
1. 发送 1 条消息
2. 快速刷新页面 5 次
3. 检查消息只显示 1 次
4. 打开 Chrome DevTools → Memory，检查内存占用
```

---

#### 4. 接龙进度实时更新
- [ ] 有人领取红包后，进度条立即更新（< 50ms）
- [ ] 显示正确的累计金额（totalClaimed）
- [ ] 显示正确的剩余金额（remainToThreshold）
- [ ] 超过阈值时，弹出警告并禁用领取按钮

**测试方法**：
```
1. 创建接龙红包（阈值 380 USDT）
2. 用户 B 领取 20 USDT
3. 观察用户 A 的进度条是否从 0 变为 20
4. 继续领取直到接近 380
5. 尝试超过 380，检查是否被拒绝
```

---

#### 5. 红包账单实时更新
- [ ] 每次有人领取，账单列表实时追加
- [ ] 显示正确的领取者（username + avatar）
- [ ] 显示正确的金额
- [ ] 账单按时间倒序排列（最新的在上面）

**测试方法**：
```
1. 用户 B 和用户 C 依次领取红包
2. 观察账单列表是否实时追加 2 条记录
3. 检查领取者和金额是否正确
4. 检查顺序是否最新在上
```

---

#### 6. 抢红包结果弹窗
- [ ] 点击领取后，立即弹出结果弹窗（< 50ms）
- [ ] 显示正确的领取金额
- [ ] 显示正确的祝福语
- [ ] 显示正确的发送者
- [ ] 显示正确的余额
- [ ] 点击遮罩层或按钮可以关闭弹窗

**测试方法**：
```
1. 用户 B 点击领取红包
2. 观察是否立即弹出弹窗
3. 检查弹窗中的数据是否正确
4. 点击"开心收下"按钮，检查弹窗是否关闭
```

---

#### 7. 余额更新
- [ ] 领取红包后，余额立即增加
- [ ] 发红包后，余额立即减少
- [ ] 余额更新延迟 < 100ms
- [ ] 只推送给当前用户（隐私保护）

**测试方法**：
```
1. 记录用户 B 领取前的余额（例如 500 USDT）
2. 用户 B 领取 20 USDT
3. 观察余额是否变为 520 USDT
4. 检查用户 A 是否收不到用户 B 的余额变动
```

---

#### 8. 网络断开重连
- [ ] 手动断开 WebSocket 后，自动重连
- [ ] 重连后同步缺失消息
- [ ] 消息顺序正确
- [ ] 无重复消息

**测试方法**：
```
1. 打开 Chrome DevTools → Network → WS
2. 右键 WebSocket 连接 → Disconnect
3. 等待 10 秒
4. 观察是否自动重连
5. 发送消息，检查是否正常
```

---

#### 9. 速率限制
- [ ] 每秒发送超过 5 条消息时，返回 429 错误
- [ ] 前端显示"发送过于频繁"提示
- [ ] 等待 1 秒后可以继续发送

**测试方法**：
```
1. 快速连续点击发送按钮 10 次
2. 观察前 5 条是否成功
3. 观察第 6 条开始是否失败
4. 等待 1 秒后，再次发送，检查是否成功
```

---

#### 10. 权限验证
- [ ] 非群成员尝试发送消息，返回 403 错误
- [ ] 非好友尝试私聊，返回 403 错误
- [ ] 前端显示友好的错误提示

**测试方法**：
```
1. 使用非群成员账号尝试发送消息
2. 观察是否返回 403 错误
3. 检查前端是否显示"您不是该群组成员"
```

## 🔒 安全验收标准

- [ ] XSS 防护生效（`<script>` 标签被过滤）
- [ ] 消息长度限制生效（最大 2000 字符）
- [ ] 速率限制生效（每秒最多 5 条）
- [ ] 权限验证生效（非成员无法发送）
- [ ] 余额隐私保护（只推送给自己）
- [ ] 无敏感信息泄露（控制台无 token、密码等）

**测试方法**：
```
1. 发送包含 <script>alert('XSS')</script> 的消息
2. 观察是否被过滤
3. 发送超过 2000 字符的消息
4. 观察是否被截断或拒绝
```
