# 红包领取接口响应规范

## 核心原则
- **后端负责计算**：累计金额、余额变动等所有数据由后端计算
- **前端负责展示**：前端只展示后端返回的数据，不做任何计算

---

## 红包领取接口响应格式

### 成功响应示例

```json
{
  "success": true,
  "data": {
    "redPacketId": "rp_xxx",
    "amount": 10.5,
    "message": "恭喜发财，大吉大利",
    "newBalance": 759730,
    "totalReceived": 125.8,
    "wasKicked": false
  }
}
```

### 被踢出响应示例

```json
{
  "success": true,
  "data": {
    "redPacketId": "rp_xxx",
    "amount": 10.5,
    "message": "恭喜发财，大吉大利",
    "newBalance": 759730,
    "totalReceived": 1000,
    "wasKicked": true,
    "kickReason": "达到领取上限"
  }
}
```

---

## 必填字段说明

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `success` | Boolean | 请求是否成功 | `true` |
| `data.redPacketId` | String | 红包ID | `"rp_xxx"` |
| `data.amount` | Number | 本次领取金额 | `10.5` |
| `data.newBalance` | Number | 领取后的用户余额 | `759730` |
| `data.totalReceived` | Number | **用户累计领取金额**（重要） | `125.8` |
| `data.wasKicked` | Boolean | 是否被踢出群组 | `false` |
| `data.kickReason` | String | 被踢出原因（可选） | `"达到领取上限"` |

## WebSocket 推送事件规范

### 问题现状
当前后端可能同时发送多个事件：
- `receiveMessage` - 通用消息事件
- `groupRedPacket` - 红包专用事件

**这会导致前端收到重复消息，触发去重逻辑。**

### 解决方案
**后端只发送一个标准事件：`receiveMessage`**

#### 事件数据结构

```json
{
  "_id": "msg_xxx",
  "groupId": "group_123",
  "sender": {
    "_id": "user_456",
    "username": "张三"
  },
  "type": "redpacket",
  "redPacketId": "rp_xxx",
  "totalAmount": 100,
  "count": 10,
  "message": "恭喜发财，大吉大利",
  "isChainRedPacket": true,
  "createdAt": "2026-04-09T12:00:00.000Z"
}
```

#### 关键字段说明

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| `_id` | String | 消息ID（用于去重） | ✅ |
| `groupId` | String | 群组ID | ✅ |
| `sender._id` | String | 发送者ID | ✅ |
| `type` | String | 消息类型：`"redpacket"` 或 `"text"` | ✅ |
| `redPacketId` | String | 红包ID（仅红包消息） | 红包必填 |
| `totalAmount` | Number | 红包总金额（仅红包消息） | 红包必填 |
| `count` | Number | 红包个数（仅红包消息） | 红包必填 |
| `isChainRedPacket` | Boolean | 是否为接龙红包 | 建议 |
| `createdAt` | String | 创建时间（ISO格式） | ✅ |

## 前端处理逻辑

### 1. 监听 `receiveMessage` 事件

```javascript
socket.on('receiveMessage', (data) => {
  const messageData = Array.isArray(data) ? data[0] : data
  
  if (messageData && messageData.groupId) {
    // 判断是否是红包
    const isRedPacket = messageData.type === 'redpacket' || 
                       messageData.redPacketId ||
                       messageData.isChainRedPacket
    
    const messageType = isRedPacket ? 'groupRedPacket' : 'group'
    handleIncomingMessage(messageType, messageData)
  }
})
```

### 2. 持久化到 LocalStorage

```javascript
// 所有通过 handleIncomingMessage 处理的消息都会自动持久化
saveToStorage(conversationId, conversations[conversationId])
```

### 3. 展示累计金额

```javascript
// 从后端响应中直接获取，不前端计算
if (response.data?.totalReceived !== undefined) {
  chainGroupInfo.value.memberInfo.totalReceived = response.data.totalReceived
}
```

## 后端需要修改的内容

### 1. 统一事件名称
- ❌ 删除：同时发送 `receiveMessage` 和 `groupRedPacket`
- ✅ 保留：只发送 `receiveMessage` 事件

### 2. 确保消息包含 `_id` 字段
- 用于前端去重，避免重复显示

### 3. 红包领取响应必须包含 `totalReceived`
- 后端计算用户累计领取金额
- 前端直接使用，不做累加

### 4. 红包领取响应必须包含 `newBalance`
- 后端计算用户最新余额
- 前端直接更新显示

## 常见问题

### Q1: 为什么前端不做累计金额计算？
**A:** 
- 前端可能被篡改，计算结果不可信
- 多端登录时，前端无法同步状态
- 后端是唯一可信的数据源

### Q2: 为什么要统一事件名称？
**A:**
- 避免前端收到重复消息
- 简化前端逻辑，不需要判断多个事件
- 便于后续维护和扩展

### Q3: 如果后端忘记返回 `totalReceived` 怎么办？
**A:**
- 前端会输出警告日志：`⚠️ 后端未返回 totalReceived，累计金额未更新`
- 累计金额不会更新，但不会影响其他功能
- 需要后端修复接口
