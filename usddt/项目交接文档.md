## 🚀 部署流程

### 1. 前端部署 (usddt)

#### 本地构建
```powershell

```

#### 打包压缩
```powershell

```

#### 上传到服务器

```bash

sudo rm -f usddt-frontend.zip ai-collaboration-hub.html collaboration-board.html
```

## 📝 开发规范

### 1. 代码修改原则
- **优先复用现有模板**，不要重新发明轮子
- **以接龙群页面为标准**，其他页面对齐
- **一次性修改完再构建上传**，不要改一个传一个

### 2. iOS 适配要点
参考 `d:\weibo\usddt\说明.txt` 中的 4 套方案：
1. focus 延迟 + scrollIntoView（最常用）
2. 监听 visualViewport（最精准）
3. fixed 底部 + 动态 bottom
4. viewport 配置（必须）

**推荐组合**: 方案4 + 方案1

### 3. 底部导航栏高度计算
- 输入框区域: ~60px
- 工具栏: ~65px
- 底部导航: ~50px
- **总计**: ~175-180px

消息列表的 `padding-bottom` 应设置为:
```css
padding-bottom: calc(16px + 180px + env(safe-area-inset-bottom));
```

## 💡 经验总结

### 成功实践
1. **iOS 安全区适配**: `viewport-fit=cover` + `env(safe-area-inset-bottom)`
2. **消息列表滚动**: `flex: 1` + `overflow-y: auto` + 足够的 `padding-bottom`
3. **后端服务管理**: 使用 PM2 管理 Node.js 进程
4. **前端部署**: 压缩后 SCP 上传，服务器上解压

### 踩坑记录
1. ❌ 不要每个页面都写底部导航栏，应该全局统一管理
2. ❌ 消息列表的 `padding-bottom` 要足够大（180px），否则会遮挡
3. ❌ iOS 键盘弹出不会改变 `window.innerHeight`，需要用 `visualViewport` 或 `scrollIntoView`
4. ❌ SCP 上传前确保目标目录存在
5. ❌ 后端服务启动需要切换到 backend 目录
