3. 最关键：解决 “键盘弹起挡住底部导航”
iPhone 键盘弹起时，fixed 底部导航会被键盘盖住，这是系统行为，我们只需要加一句 CSS 就能解决：
css
/* 全局 CSS 加这个 */
body {
  height: 100%;
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  bottom: 0;
  padding-bottom: env(safe-area-inset-bottom);
}

在 Vue 3 + iPhone 场景下，底部固定输入框被键盘挡住，核心原因是：iOS Safari 键盘弹出时，不会压缩视口（window.innerHeight 不变），而是直接覆盖在页面上，导致 fixed 在底部的输入框被遮住。
下面给你 4 套可直接复制的 Vue 3 方案（从简单到稳定），优先用 方案 1 + 方案 4 组合。
方案 1：focus 延迟 + scrollIntoView（最常用、最简单）
原理：输入框聚焦后，延迟 300ms 等键盘动画完成，再把输入框滚到可视区。
vue
<template>
  <div class="page">
    <!-- 内容区 -->
    <div class="content" />

    <!-- 底部输入框 -->
    <div class="footer-input">
      <input
        type="text"
        placeholder="输入消息"
        @focus="handleFocus"
        @blur="handleBlur"
      />
    </div>
  </div>
</template>

<script setup>
const handleFocus = (e) => {
  // 延迟 300ms 等键盘弹出
  setTimeout(() => {
    e.target.scrollIntoView({
      behavior: 'smooth',
      block: 'nearest', // 就近滚动，不强行顶到顶部
      inline: 'nearest'
    })
  }, 300)
}

const handleBlur = () => {
  // 可选：键盘收起时复位滚动
  window.scrollTo({ top: 0, behavior: 'smooth' })
}
</script>

<style scoped>
.page {
  height: 100%;
}
.content {
  height: calc(100% - 60px);
  overflow-y: auto;
}
.footer-input {
  position: fixed;
  left: 0;
  bottom: 0;
  width: 100%;
  height: 60px;
  background: #fff;
  padding: 10px 16px;
  box-sizing: border-box;
  /* 适配 iPhone 底部横条 */
  padding-bottom: calc(10px + env(safe-area-inset-bottom));
}
input {
  width: 100%;
  height: 40px;
  border-radius: 20px;
  border: 1px solid #eee;
  padding: 0 12px;
}
</style>
方案 2：监听 visualViewport（iOS 最精准）
原理：用 window.visualViewport 监听键盘高度，动态给 body 加下内边距，把页面顶上去。
vue
<script setup>
import { onMounted, onUnmounted } from 'vue'

onMounted(() => {
  const visualViewport = window.visualViewport
  if (!visualViewport) return

  const handleResize = () => {
    const offset = visualViewport.offsetTop || 0
    // 键盘高度 = 原始高度 - 当前可视高度
    const keyboardHeight = window.innerHeight - visualViewport.height
    document.body.style.paddingBottom = `${keyboardHeight + offset}px`
  }

  visualViewport.addEventListener('resize', handleResize)
  visualViewport.addEventListener('scroll', handleResize)

  onUnmounted(() => {
    visualViewport.removeEventListener('resize', handleResize)
    visualViewport.removeEventListener('scroll', handleResize)
    document.body.style.paddingBottom = ''
  })
})
</script>
方案 3：fixed 底部 + 动态 bottom（聊天页常用）
原理：把固定 bottom:0 改成动态 bottom，键盘弹出时抬高输入框。
vue
<template>
  <div class="footer-input" :style="{ bottom: keyboardHeight + 'px' }">
    <input type="text" @focus="handleFocus" @blur="handleBlur" />
  </div>
</template>

<script setup>
import { ref } from 'vue'
const keyboardHeight = ref(0)

const handleFocus = () => {
  setTimeout(() => {
    // 估算 iPhone 键盘高度（可根据机型微调）
    keyboardHeight.value = /iPhone/.test(navigator.userAgent) ? 280 : 240
  }, 300)
}

const handleBlur = () => {
  keyboardHeight.value = 0
}
</script>

<style>
.footer-input {
  position: fixed;
  left: 0;
  width: 100%;
  transition: bottom 0.2s ease; /* 平滑动画 */
}
</style>

/* style.css */

  /* 兼容底部横条 */
  padding-bottom: env(safe-area-inset-bottom);
}
常见坑与避坑
不要用 100vh
iOS 键盘弹出时 100vh 不变，会导致布局错乱。
css
/* ❌ 坏 */
.page { height: 100vh; }
/* ✅ 好 */
html, body { height: 100%; }
.page { height: 100%; }
fixed 父级不要有 transform
父元素有 transform 会让 fixed 相对于父级而非视口。
scrollIntoView 必须延迟
直接在 focus 调用会被键盘动画覆盖，必须 setTimeout 300ms。
推荐组合（最稳）
页面加 方案 4（viewport + safe-area）
输入框用 方案 1（focus 延迟 scrollIntoView）
聊天 / 长列表页再加 方案 2（visualViewport 监听）
